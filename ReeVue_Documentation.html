ReeVue

A community-driven tech career review platform where engineers discover, share, and evaluate real interview experiences and workplace insights across top technology companies.

⚛ React 19 🔥 Firebase 12 ⚡ Vite 8 Bootstrap 5 React Router 7 Cloud Firestore
1

Project Overview

ReeVue is a web application enabling engineers and tech professionals to discover, share, and evaluate interview experiences and job reviews at major technology companies. The platform functions as a community-driven resource that helps users make informed career decisions based on authentic peer experiences.

💡

Core idea: Aggregated, anonymous feedback from real candidates — giving job seekers an honest picture before they walk into an interview room.

Core Features

  • Anonymous interview feedback from verified candidates at top tech companies
  • Aggregated company ratings with difficulty assessments for full interview loops
  • Community Q&A forum for technical and behavioral interview preparation
  • Personalized profiles tracking user contributions, badges, and saved companies

Target Audience

  • Computer science students preparing for tech industry interviews
  • Software engineers evaluating career opportunities at new companies
  • Tech professionals looking to share and document workplace experiences

Context & State Management

ReeVue uses the React Context API for global state. Three context providers wrap the app from the outside in — no third-party state library required.

ThemeContext

Manages light/dark mode toggle. Persists preference to localStorage under reevue_theme.

AuthContext

Handles authentication, session persistence, and real-time Firestore review sync via onSnapshot.

CompaniesContext

Lightweight context for global company search filters, managed with useReducer.

Firebase is configured in src/firebase/config.js and exports three services consumed across the application.

Firestore Collections

CollectionKey FieldsDescription
reviews companyName, jobTitle, rating, difficulty, pros, cons, authorEmail, recommend, isAnonymous, createdAt, helpfulCount Primary data store. Queried in real-time via onSnapshot in AuthContext. New documents added via addDoc with serverTimestamp.

Firebase Operations

  • CRUD: getDocs, addDoc, updateDoc, deleteDoc, getDoc
  • Auth: onAuthStateChanged, signInWithEmailAndPassword, createUserWithEmailAndPassword, signOut, updateProfile
  • Real-time: onSnapshot listener on the reviews collection
  • Timestamps: serverTimestamp for consistent server-side review ordering

Pages & Components

Home Page /

The landing page is composed of three distinct sections:

  • Hero Section: Full-viewport background image with light/dark variant, headline, and CTAs. Background switches based on isDark from ThemeContext.
  • Company Directory: Dynamically built from Firestore review data. Groups reviews by company, calculates average rating and modal interview difficulty. Each card navigates to /company/:name.
  • Highest Rated Reviews: Fetches all reviews, sorts by rating descending, displays the top 2 using ReviewCard.

Reviews Page /reviews

A filterable, sortable list of all reviews synced from Firestore into localStorage.

  • Filter by interview difficulty: All, Easy, Medium, Hard
  • Sort by: Most Helpful, Highest Rated, Newest First
  • Text search across companyName, jobTitle, pros, and cons fields
  • Review owners see Edit and Delete action buttons on their ReviewCard components

Submit Review /submit-review Protected

A 5-step guided wizard for submitting or editing a review:

1
Company Profile
Company Name, Industry Sector
2
Role Specifications
Job Title, Employment Type, Location City
3
Workspace Insights
Pros (15+ chars), Cons (15+ chars)
4
Experience Rating
Star Rating (1–5), Difficulty (Easy / Medium / Hard), Recommend toggle
5
Review & Publish
Live preview, anonymous posting toggle, submit or save draft
✓

Drafts auto-save to reevue_review_draft in localStorage. Edit mode loads existing review data from Firestore by editingReviewId passed via router state.

Company Profile /company/:name

Reads the company name from URL params, loads all reviews from localStorage, then renders CompanyDetails — which handles: aggregated rating summary, difficulty breakdown, per-review list filtered to that company, and a Write Review CTA.

Search /search

Receives a query string via ?q= URL parameter. Filters reviews across companyName, jobTitle, pros, and cons fields. Also computes matching company cards with aggregated stats. Authenticated users can edit/delete their own reviews inline.

Interview Q&A /qa Protected

  • Questions stored in localStorage (reevue_qa_v1) with seed data for Google, Stripe, Meta, Amazon
  • Department filter: All, Engineering, Product, Design, Operations, Other
  • Accordion-style expand/collapse per question
  • Add Question form: company name, department, and question text (10+ chars)
  • Inline answer input under each expanded question

User Profile /profile Protected

A four-tab personal dashboard:

  • My Reviews: All reviews authored by the logged-in user. Supports inline edit and delete.
  • Saved Companies: Bookmarked companies loaded from reevue_saved_companies.
  • My Q&A: Q&A questions submitted by the user or with community answers.
  • Settings: Name and bio edits synced to AuthContext + Firebase. Notification preferences (UI only).

Achievement badges are dynamically awarded based on review count and helpful vote thresholds.

Login & Sign Up /login /signup

Both pages hide the Navbar and Footer via an isLoginPage check in App.jsx. Login performs email format and 8-character password validation before calling Firebase. Sign Up includes a password strength meter (Weak / Fair / Good / Strong) and optional phone and bio fields.

Authentication System

ReeVue implements a full Firebase Authentication flow with persistent sessions. The ProtectedRoute guard ensures all sensitive routes redirect unauthenticated visitors.

Sign Up Flow

  • User fills in first name, last name, email, optional phone, password (8+), confirm password, optional bio
  • Client-side validation runs before any network call
  • createUserWithEmailAndPassword creates the Firebase account
  • updateProfile sets the displayName on the Firebase Auth user
  • onAuthStateChanged fires, formats the user object, persists to localStorage
  • User is redirected to the home page

Login Flow

  • Email format and password length validated client-side before any network call
  • signInWithEmailAndPassword called with Firebase Auth
  • On success, onAuthStateChanged updates context state
  • User is redirected to the home page

Protected Route Guard

ProtectedRoute is an Outlet wrapper that reads isLoggedIn from AuthContext. If false, it renders <Navigate to="/login" replace />. All four protected routes are nested under a single ProtectedRoute element in the routes tree.

Logout

  • Removes reevue_auth_v1 from localStorage
  • Sets user state to null
  • Calls Firebase signOut()
  • Navigates to the home page

Getting Started

Prerequisites

  • Node.js 18+ and npm
  • A Firebase project with Authentication (Email/Password) and Firestore enabled

Installation

cd DEPI_REACT-main

# 2. Install dependencies
npm install

# 3. Configure Firebase
#    Edit src/firebase/config.js with your Firebase project credentials

# 4. Start development server
npm run dev

# 5. Build for production
npm run build
